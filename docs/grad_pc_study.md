# 可微分梯度增强预测-校正耦合:精确梯度证伪研究(完整报告)

**研究问题**(用户提出):Phase A 的负结果是否仅因方法粗糙?引入真·可微分仿真(精确导数)后,"可微分+预测-校正"能否在速度/精度上超越不可微版本?

**回答:不是方法粗糙。我们把每一级近似都替换到机器精度的精确导数(复步微分,vs FD 验证收敛至 5.8e-11),结果不变或更差。"粗糙"假说在最强证据标准下被证伪;失败是机制性的。**

## 方法:五级导数质量阶梯

对校正遍的窗内力插值(已诊断的主导误差项),逐级提升端点导数 dF/dβ 的质量:

| 级 | 方法 | 每块额外成本 |
|---|---|---|
| L0 linear | 无导数(基线=MATLAB) | 0 |
| L1 quad | 历史三点抛物线(F_{k-1},F_k,F_{k+1}) | 0 |
| L2 march-FD | 保持力前瞻 2 步 + 全流体解差分 | +4 解 |
| L3 traj-FD | 轨迹切向中心差分(冻结尾迹力映射) | +4 冻结解 |
| L4 **exact-CS** | **复步微分 = 机器精度前向 AD**(全链复数安全化,实数行为逐位不变;CS vs FD 收敛 3.7e-7→5.8e-11) | +2 复数解 |

对照组(不可微):picard2/picard3(窗口不动点重迭代)。

关键数值事实:**traj-FD 与 exact-CS 在所有 M* 下一致到 5 位数**——FD 早已精确测得冻结映射方向导数;精确化不改变任何结论。

## 机制(为什么端点导数救不了重载)

1. **微分对象错误,且无可负担的正确对象**:Hermite 需要全耦合传播子沿校正轨迹的 dF/dt(含结构推进+附加质量反作用+尾迹演化)。可负担的梯度(冻结尾迹力映射 JVP)即使精确也缺失耦合贡献;微分全耦合传播子的成本 ≥ 它要取代的 Picard 迭代。
2. **重载下端点斜率巨大**(附加质量主导 dF/dX),近似切向的小误差被放大,三次 Hermite 端点过陡 → 窗内振荡发散(51%)。
3. **march-FD 在 M*=0.4 的"胜利"是假象**:其前瞻含全流体解(含尾迹),偶然采样了部分真耦合响应——有效的是**采样耦合映射**,不是**微分部分映射**。这解释了为何更"准"的导数反而更差。
4. 一致赢家:**迭代采样**(picard2,均匀最优)与**零成本历史曲率**(quad,重载 2.2×)。

## 工程结论(头对头判决)

- **耦合循环内部**:可负担的可微分增强**不优于**不可微替代。重载选 picard2(+1 解+1 重推进),预算紧选 quad(免费 2.2×);轻载维持基线。
- **可微分仿真的价值通道在外环**:设计/控制伴随梯度(一次伴随摊销整条轨迹)——Warp tape 已验证(1e-7~1e-11)。
- 方法论资产:复数安全化精确力链(实数逐位不变)是后续梯度工作的基础设施;五级阶梯协议可复用于其他耦合方案的"粗糙假说"检验。
